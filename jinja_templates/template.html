Mass Balance calibration continuous integration

This website tracks the performance of the OGGM mass balance calibration across multiple model versions. This allows us to easily see the effects of certain model changes on the mass balance parametrisation of OGGM.

How to use:
The website is organised in OGGM versions. You can switch between different model Versions with the buttons labeled like 1.0.2+12.abcd345. This string indicates the model version and also the respective git-commit.

Some version-strings end with -histalp. Only Alpine reference glaciers were used for these cross-validations and instead of the CRU climate data, the ZAMG Histalp data was used. The cross-validation with the Histalp data also allowed us to propose a set of standard model parameters to be used with this data.

The general overview of each model version first provides a histogram comparing two different approaches to determine the glaciers temperature sensitivity parameter. For more information on this see The Open Global Glacier Model, Figure 4.

Additionally the overview also shows results for the cross-validation when varying the model parameters Precipitation Factor, Melt Temperature, Liquid Precipitation Temperature and Temperature Laps Rate. Each possible parameter combination (1215) is cross-validated and plotted with respect to one of the parameters. The box-whisker plots show the full range of results and the red dots indicates the results when useing the proposed standard values.

With Select glacier you can search for and select a specific glacier and see how the mass balance model performs in its case. The plots compare the measured annual mass balance on this glacier with two different OGGM results: For the Calibrated t_star the annual mass balance is used to calibrate the glaciers temperature sensitivity in a way that the resulting mass balance bias is 0. The Crossvalidated t_star does not use the measured mass balance data and therefore represents true model performance.
